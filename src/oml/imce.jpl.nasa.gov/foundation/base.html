Abstract

The Base vocabulary defines foundation concepts that are common to multiple specialized ontologies.

AggregatedElement is the domain and rage of the aggregates property. Aggregation is a homeomeric relation that is potentially many-to-many. 5. Relation Entities

5.1. Aggregates

aggregates is a homeomeric many-to-many relation used to represent relations corresponding to directed acyclic graphs. Context-specific aggregation relations should specialize aggregates.

Source AggregatedElement
Target AggregatedElement
Forward Relation aggregates
Reverse Relation isAggregatedIn
Attributes
  • Asymmetric
  • Irreflexive

contains is a homeomeric one-to-many relation used to represent tree structures. Context-specific containment relations should specialize contains.

6. Scalar Properties

6.1. hasAlternateName

"hasAlternateName property relates an IdentifiedElement to zero or more alternate names. Alternate names need not be unique. Context-specific alternate names should specialize hasAlternateName.

Domain IdentifiedElement
Range string

6.2. hasCanonicalName

"hasCanonicalName relates an IdentifiedElement to zero or one standard or official names. Canonical names need not be unique. Context-specific canonical names should specialize hasCanonicalName.

Domain IdentifiedElement
Range string

6.3. hasContainmentIndex

"hasContainmentIndex indicates the unique position of a ContainedElement in a tree of contains relationships. One intended use for hasContainmentIndex is to indicate order of siblings in a serial presentation such as a graph or indented list. Context-specific containment indices (e.g., hierarchical work breakdown structure numbers) should specialize hasContainmentIndex.

Domain ContainedElement
Range string

6.4. hasDescription

"hasDescription relates an IdentifiedElement to zero or one text descriptions. Context-specific descriptions should specialize hasDescription.

Domain IdentifiedElement
Range string
Attributes Functional

6.5. hasIdentifier

"hasIdentifier property relates an IdentifiedElement to zero or more identifiers. Context-specific identifiers should specialize hasIdentifier.

Domain IdentifiedElement
Range string

6.6. hasIndexEntry

hasIndexEntry relates an IdentifiedElement to zero or one terms under which it would be indexed in a document. Context-specific index entries (e.g., a DocBook indexentry ) should specialize hasIndexEntry.

Domain IdentifiedElement
Range string

6.7. hasShortName

hasShortName relates an IdentifiedElement to zero or more short names. Short names need not be unique. Context-specific short names (e.g., assembly name) should specialize hasShortName

Domain IdentifiedElement
Range string

6.8. hasSortKey

hasSortKey relates an IdentifiedElement to zero or one string-valued keys that may be used to establish its order in a sorted collection. Context-specific sort keys should specialize hasSortKey.

Domain IdentifiedElement
Range string

6.9. hasUuid

hasUuid relates an IdentifiedElement to zero or one universally unique identifiers. Context-specific unique identifiers should specialize hasUuid.

Domain IdentifiedElement
Range string
