Abstract

The Analysis Ontology defines general concepts and properties for analyses (e.g., trade studies, driving requirements analysis, etc.). It provides a basis for specialization by domain experts.

4. Concepts

4.1. Analysis

Analysis is a deprecated synonym for Explanation .

Supertypes: Explanation

Subtypes: DrivingRequirementsExplanation, KeyRequirementsExplanation, TradeStudy

4.2. Assumption

An Assumption is Characterization that is taken to be true for the purpose of analysis.

Supertypes: Characterization

Subtypes: Fact

Relations with source:

  • Assumption â Limits â CharacterizedElement

4.3. Characterization

A Characterization is an element that provides a parametric or other characterization of another model element. The canonical example of a Characterization is a set of name/value pairs, but other characterizations (e.g., classification) are possible.

Supertypes: AnalyzedElement, CharacterizedElement, base:IdentifiedElement

Subtypes: Assumption, project:DevelopmentState, Explanation, GlossaryEntry, metrology:Quantity

Relations with source:

  • Characterization â Characterizes â CharacterizedElement

4.4. Constraint

A Constraint is a container for one or more mathematical expressions asserting constraints on values.

Supertypes: AnalyzedElement, CharacterizedElement

Relations with source:

  • Constraint â Constrains â ConstrainedElement

4.5. Criterion

A Criterion establishes a region for some Metric that corresponds to success for some MeasuredElement . A Criterion for the Metric sample mass in kg might be at least 10 kg .

Supertypes: AnalyzedElement, CharacterizedElement, base:IdentifiedElement

4.6. DrivingRequirementsAnalysis

DrivingRequirementsAnalysis is a deprecated synonym for DrivingRequirementsExplanation .

Supertypes: DrivingRequirementsExplanation

4.7. DrivingRequirementsExplanation

A DrivingRequirementsAnalysis provides the rationale for designating a set of Requirement s as driving , and makes the designation via the explains property.

Supertypes: Analysis

Subtypes: DrivingRequirementsAnalysis

Relations with source:

  • DrivingRequirementsExplanation â Explains â mission:Requirement

4.8. Explanation

An Explanation is a product that captures or summarizes the results of an analysis activity and relates it to one or more other model elements. It may contain narrative prose directly or provide a reference to external products. Driving requirements analyses, trade studies, and cost estimates are all examples of Explanation .

Supertypes: base:AggregatedElement, Characterization

Subtypes: Analysis

Relations with source:

  • Explanation â Analyzes â AnalyzedElement

  • Explanation â Explains â CharacterizedElement

  • Explanation â Validates â CharacterizedElement

4.9. Fact

A Fact is an Assumption that is true.

Supertypes: Assumption

4.10. GlossaryEntry

GlossaryEntry is a Characterization that associates a term with a definition.

Supertypes: Characterization

Properties:

  • hasGlossaryDefinition

  • hasGlossaryTerm

4.11. KeyRequirementsAnalysis

KeyRequirementsAnalysis is a deprecated synonym for KeyRequirementsExplanation .

Supertypes: KeyRequirementsExplanation

4.12. KeyRequirementsExplanation

A KeyRequirementsAnalysis provides the rationale for designating a set of Requirement s as key , and makes the designation via the explains property.

Supertypes: Analysis

Subtypes: KeyRequirementsAnalysis

Relations with source:

  • KeyRequirementsExplanation â Explains â mission:Requirement

4.13. TradeStudy

A TradeStudy explains some set of model elements and their properties through comparison with alternatives.

Supertypes: Analysis

5. Relation Entities

5.1. Analyzes

Attributes: Asymmetric, Irreflexive

Source: Explanation

Target: AnalyzedElement

Forward Relation: analyzes An Explanation a analyzes some IdentifiedElement e if and only if a considers or otherwise takes account of e .

Reverse Relation: isAnalyzedBy

Supertypes: AnalyzedElement, CharacterizedElement

5.2. Characterizes

Attributes: Asymmetric, Irreflexive

Source: Characterization

Target: CharacterizedElement

Forward Relation: characterizes A Characterization c characterizes a CharacterizedElement e if and only if c describes, delimits, or restricts e .

Reverse Relation: isCharacterizedBy See characterizes .

Supertypes: AnalyzedElement, CharacterizedElement, mission:SpecifiedElement

Subtypes: Explains, metrology:IsQuantityOf, Limits

5.3. Constrains

Attributes: Asymmetric, Irreflexive

Source: Constraint

Target: ConstrainedElement

Forward Relation: constrains A Constraint c constrains some ConstrainedElement e if and only if c imposes restrictions on one or more properties of e .

Supertypes: AnalyzedElement, CharacterizedElement

5.4. Explains

Attributes: Asymmetric, Irreflexive

Source: Explanation

Target: CharacterizedElement

Forward Relation: explains An Explanation a explains some SpecifiedElement e if and only if a provides a rationale or justification for some design aspect of e .

Reverse Relation: isExplainedBy See explains .

Supertypes: AnalyzedElement, CharacterizedElement, Characterizes

Subtypes: Validates

5.5. Limits

Attributes: Asymmetric, Irreflexive

Source: Assumption

Target: CharacterizedElement

Forward Relation: limits

Reverse Relation: isLimitedBy See limits .

Supertypes: AnalyzedElement, CharacterizedElement, Characterizes

5.6. Validates

Attributes: Asymmetric, Irreflexive

Source: Explanation

Target: CharacterizedElement

Forward Relation: validates An Explanation a validates some SpecifiedElement e if and only if a provides a definitive, determinative explanation for some design aspect of e .

Reverse Relation: isValidatedBy See validates .

Supertypes: AnalyzedElement, CharacterizedElement, Explains
