Abstract

This vocabulary is a partial account of Prof. Rene Dybkaerâs 'An Ontology of Property' focused on the concepts necessary for dimensional analysis of systems of metrological units and of systems of quantities based on the algebraic relationships involved in the definitions of the metrological units and the kinds of quantities in these respective systems.

See the rationale behind the 3rd revision of the Vocabulary of International Metrology, VIM3, for references to Prof. Dybkaerâs influential work: https://www.bipm.org/en/publications/guides/rationale-vim3.html

Where possible, correspondences to VIM3 definitions will be referenced from https://jcgm.bipm.org/vim/en/index.html The VIM3 is available in document PDF here: https://www.bipm.org/utils/common/documents/jcgm/JCGM_200_2012.pdf

References to definitions in the ontology are indicated via a reference in parentheses with a chapter and section and a page number; for example, (S.2.23, p. 33) refers to the definition of object on page 33.

3.6. SystemOfUnitaryKindsOfQuantities

A metrology:SystemOfUnitaryKindsOfQuantities is a collection of metrology:UnitaryKindOfQuantity(ies) that can be partitioned into two groups: base and derived. Note that this partitioning is independent of how the algebraic definition of each metrology:UnitaryKindOfQuantity (i.e metrology:IndependentUnitaryKindOfQuantity or metrology:DerivedUnitaryKindOfQuantity).

Applying dimensional analysis to a metrology:SystemOfUnitaryKindsOfQuantities means applying the rules of dimensional calculus to the alegbraic definitions required for expressing each derived metrology:UnitaryKindOfQuantity of the system as a product of powers involving only the base metrology:UnitaryKindOfQuantity of the system.

Supertypes: MetrologicalEntity

Relations with domain:

  • SystemOfUnitaryKindsOfQuantities systemHasUnitaryKindOfQuantity UnitaryKindOfQuantity

4.4. Object

A metrology:Object is anything perceivable or conceivable.

A metrology:Object serves two purposes: 1) Specify the subject of a metrology:Quantity relationship with a metrology:QuantityValue. 2) Provide a reference for elaborating a rich description elsewhere using an appropriate vocabulary.

Note: Dybkaerâs ontology defines additional vocabulary about object (S.2.24) with the subconcepts of system (S.3.3) and component (S.3.4), which are further divided in terms of element, relationship, and process (see Figure 3.5, p. 41).

Since none of the subconcepts of object (S.2.23) provide any additional semantics to the metrology vocabulary, they are omitted here. This ommision simplifies the task of establishing correspondences between metrology:Object in this vocabulary and rich descriptions elsewhere constructed using an appropriate vocabulary.

Supertypes: Thing

Relations with range:

  • Quantity isQuantityOf Object

4.7. Quantity

A metrology:Quantity is an inherent state- or process-descriptive feature of a metrological:Object that has a magnitude expressed as a metrological:QuantityValue.

This definition is adapted from the expanded definition of quantity (S.12.13, S.12.14).

The definition of metrology:Quantity as an OML aspect provides significant flexibility for modeling. 1) The relationship metrology:IsQuantityOf between a metrology:Quantity and a metrology:Object can be specified explicitly (i.e., asserted) or implicitly via axioms contraining metrology:isQuantityOf. 2) The relationship metrology:HasMagnitude between a metrology:Quantity and a metrology:QuantityValue can be specified explicitly (i.e., asserted) or implicitly via axioms constraining metrology:hasMagnitude. 3) metrology:Quantity and metrology:QuantityValue can be modeled as distinct entities or the same entity.

Supertypes: MetrologicalEntity

Subtypes: OrdinalQuantity, UnitaryQuantity

Relations with domain:

  • Quantity hasKindOfQuantity KindOfQuantity

  • Quantity hasMagnitude QuantityValue

  • Quantity isQuantityOf Object

4.15. Prefix

A metrology:Prefix is an opaque representation of a multiplier used for defining a metrology:PrefixedMeasurementUnit. This representation facilitates dimensional analysis of a metrology:SystemOfMetrologicalUnits as a logical analysis since prefixes are individuals. Had the prefix multipler been defined as a scalar property typed by a datatype, dimensional analysis would be a more complicated analysis involving a combination of logical and numerical analysis techniques.

Supertypes: MetrologicalEntity

Subtypes: 80000-1-decimalPrefix:DecimalPrefix

Relations with range:

  • PrefixedMeasurementUnit hasPrefix Prefix

Although every metrology:Quantity must have exactly one metrology:Object as its metrology:isQuantityOf, this relationship, metrology:isQuantityOf, can be specified either explicitly as an asserted axiom or implicitly as a constraint such as a range restriction or an inference rule.

Although every metrology:Quantity must have exactly one metrology:QuantityValue as its metrology:hasMagnitude, this relationship, metrology:hasMagnitude, can be specified either explicitly as an asserted axiom or implicitly as a constraint such as a range restriction or an inference rule.

Attributes:

Source: Quantity

Target: QuantityValue

Forward Relation: hasMagnitude

5.3. HasKindOfQuantity

metrology:HasKindOfQuantity establishes the associative relationship between the concepts of metrology:Quantity and metrology:KindOfQuantity. There concepts differ in two fundamental ways: 1) The concept of metrology:Quantity is intrinsically tied to a metrology:Object for which it is a metrology:isQuantityOf; there is no corresponding capability for metrology:KindOfQuantity. 2) The concept metrology:KindOfQuantity is intrinsically tied to a metrological commitment to characterize its algebraic dependence (or not) with respect to other metrology:KindOfQuantity; there is no corresponding capability for metrology:Quantity.

Attributes: Functional

Source: Quantity

Target: KindOfQuantity

Forward Relation: hasKindOfQuantity

6.2. isDimensionlessKindOfQuantity

metrology:isDimensionlessKindOfQuantity specifies whether the definition of a metrology:UnitaryKindOfQuantity has dimension one, either intrisically as an metrology:IndependentUnitaryKindOfQuantity or extrinsically as a metrology:DerivedUnitaryKindOfQuantity where the product factors of other metrology:UnitaryKindOfQuantity(ies) could be algebraically reduced to one.

Note when defining a metrology:DerivedUnitaryKindOfQuantity with metrology:isDimensionlessKindOfQuantity=true, it is important to retain the non-simplified product factors in order to preserve the metrological provenance of the metrology:DerivedUnitaryKindOfQuantity with respect to other metrology:UnitaryKindOfQuantity(ies).

Domain: UnitaryKindOfQuantity

Scalar value type: boolean

Attributes: Functional

6.3. isBaseKindOfQuantity

It is a property of a metrology:SystemOfUnitaryKindsOfQuantities to select of a subset of metrology:UnitaryKindOfQuantity as being functionally independent of each other as its base metrology:UnitaryKindOfQuantity such that the other metrology:UnitaryKindOfQuantity in the system can be algebraically derived from the base metrology:UnitaryKindOfQuantity.

Note that selecting a metrology:UnitaryKindOfQuantity as a base or derived for a given system is unrelated to its definition (i.e metrology:IndependentUnitaryKindOfQuantity or metrology:DerivedUnitaryKindOfQuantity).

Domain: SystemHasUnitaryKindOfQuantity

Scalar value type: boolean

Attributes: Functional

metrology:hasDoubleNumber provides the capability to specify the number of a metrology:QuantityValue as an IEEE 64-bit floating-point number. If this capability is inadequate, the metrology vocabulary can be extended with additional properties for an alternate representation of a number.

Domain: QuantityValue

Scalar value type: double

Attributes: Functional

6.6. isDimensionlessMeasurementUnit

metrology:isDimensionlessMeasurementUnit specifies whether the definition of a metrology:CoherentMeasurementUnit has dimension one, either intrisically as an metrology:IndependentMeasurementUnit or extrinsically as a metrology:DerivedMeasurementUnit where the product factors of other metrology:MeasurementUnit(s) could be algebraically reduced to one.

Note when defining a metrology:DerivedMeasurementUnit with metrology:isDimensionlessMeasurementUnit=true, it is important to retain the non-simplified product factors in order to preserve the metrological provenance of the metrology:DerivedMeasurementUnit with respect to other metrology:MeasurementUnit(same).

Domain: CoherentMeasurementUnit

Scalar value type: boolean

Attributes: Functional

6.7. isBaseMetrologicalUnit

It is a property of a metrology:SystemOfMetrologicalUnits to select of a subset of metrology:MeasurementUnit as being functionally independent of each other as its base metrology:MeasurementUnit(s) such that the other metrology:MeasurementUnit(s) in the system can be algebraically derived from the base metrology:MeasurementUnit(s).

Note that selecting a metrology:MeasurementUnit as a base or derived for a given system is unrelated to its definition (i.e metrology:NonCoherentDerivedMeasurementUnit, metrology:IndependentMeasurementUnit, metrology:DerivedMeasurementUnit or metrology:PrefixedMeasurementUnit).

Domain: SystemHasMetrologicalUnit

Scalar value type: boolean

Attributes: Functional
