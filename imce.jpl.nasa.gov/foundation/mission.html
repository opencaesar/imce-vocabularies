Abstract

The Mission vocabulary defines concepts and properties for describing missions in terms of their objectives, their constituent components, the functions those components perform, and the requirements that specify them.

3. Aspects

3.1. PerformingElement

PerformingElement is the domain of performs. Any concept that performs a Function should specialize PerformingElement.

Subtypes
  • Component
  • Mission
Relations
  • performs : Function
  • produces : Product

3.2. PresentingElement

PresentingElement is the domain of presents. Any concept that presents an Interface should specialize PresentingElement.

Subtypes
  • Component
  • Interface
  • Mission
Relations
  • presents : Interface

3.3. SpecifiedElement

SpecifiedElement is the range of specifies. Any concept or reified relation that isSpecifiedBy a Requirement should specialize SpecifiedElement.

Subtypes
  • analysis:Characterizes
  • base:Contains
  • Component
  • Mission
  • Performs
  • Presents
  • Produces
  • TransfersIn
  • TransfersOut
Relations
  • isSpecifiedBy : Requirement

3.4. TraversingElement

TraversingElement is the domain of traverses. Any concept that traverses an Interface should specialize TraversingElement.

Subtypes
  • Flow
  • Item
Relations
  • traverses : Junction

4. Concepts

4.1. Component

A Component is a designed entity that exists to perform a Function. Example Components include launch vehicle, spacecraft, telecommunication subsystem, flight software, attitude control software, and mission operations team.

4.2. Environment

An Environment represents a set of conditions under which a Component performs some Function. Example environments include low earth orbit, trans-Jupiter cruise, and Martian north polar surface.

Supertypes
  • base:AggregatedElement
  • base:IdentifiedElement
Relations
  • base:aggregates : Environment
  • base:isAggregatedIn : Environment
  • influences : Component
  • isInducedBy : Component

4.3. Flow

A Flow is a non-discrete transfer of substance or energy between Components. Non-discrete implies that the substance or energy has no natural boundaries. Examples of Flow include electric current, propellant, and heat.

Supertypes
  • base:ContainedElement
  • base:Container
  • base:IdentifiedElement
  • TraversingElement
Relations
  • base:contains : Flow
  • base:isContainedIn : Flow

4.4. Function

A Function is a purpose for which a PerformingElement is included in a designed system.

4.5. Interface

An Interface represents a set of mechanical, electrical, signal, or other features that describe some aspect of a PresentingElement's connection to or interaction with another PresentingElement.

4.8. MaterialItem

A MaterialItem is a discrete unit of material exchanged between two Functions. Example MaterialItems include rock hammer and soil sample.

Supertypes
  • Item
Relations
  • base:contains : MaterialItem
  • base:isContainedIn : MaterialItem

4.9. Message

A Message is a discrete unit of information exchanged between two Functions. Example Messages include command, command response, and file contents.

Supertypes
  • Item
Relations
  • base:contains : Message
  • base:isContainedIn : Message

4.10. Mission

A Mission is a PerformingElement that pursues Objectives.

Supertypes

4.11. Objective

An Objective represents a specific interest that one or more stakeholders have in the successful execution of a mission.

Supertypes
  • base:AggregatedElement
  • base:IdentifiedElement
  • project:Concern

  • base:isAggregatedIn : Objective

A Product is a tangible artifact that results from achievement of mission objectives.

Supertypes

  • base:IdentifiedElement
Relations

  • base:isAggregatedIn : Product
